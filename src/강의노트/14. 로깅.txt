[ 로깅 ]
    1. 정의 : 프로그램 실행중 발생하는 정보를 기록하는 행위
    2. 목적 : print() 대신에 로깅 사용하여 '저장' 또는 효율적인 구분(레벨)으로 유지보수
    3. 자바(스프링) 로깅 하는 방법
        권장) 롬복에서 제공한느 @Log4j2활용한 컨트롤러와 서비스에서 주로 사용된다.
        @Log4j2 class Controller{}
    4. 주요 객체/함수
        log.debug()     : 개발 및 테스트단계 사용 <개발 권장>
        log.info()      : 데이터들의 상태확인 사용.<개발/운영 권장>
        lgg.warn()      : 경고확인 사용. <운용 권장>
        log.error()     : 예외 확인 사용. <운용 권장>
    5. application.properties 파일에서 로깅 설정
        debug < info < warn < error
        1) 스프링에서는 info 가 기본값 이다.
            logging.level.root = 레벨선택
        2) 특정 패키지에서만 설정 , 패키지명은 [java]폴더 이하 경로이다.< 한글 패키지 미지원 >
            logging.level.패키지명.패키지명 = 레벨
    6. 로그파일 저장위치 설정
        logging.file.name = 폴더명/파일명.log
        logging.logback.rollingpolicy.max-file-size = 10MB  : 로그파일 용량
        logging.logback.roollingpolicy.max-history = 10     : 로그파일 최대개수
    7. 프레임워크 내부 로그 줄이기( 운영단계에서 warn 이상만 출력 )
       spring.main.log-startup-info = false
       logging.level.org.springframework.boot.web=warn
       logging.level.org.springframework.boot.web.embedded.tomcat=warn
       logging.level.org.apache.catalina=warn
       logging.level.org.apache.coyote=warn
       logging.level.org.springframework.web=warn
    8. 로그 형식
        - %msg : 로그메시지  , %n : 줄바꿈            , %d{ y M d H m s }  : 날짜/시간
        - %level : 레벨     , logger : 클래스/경로

        logging.pattern.console =
