[ react / 리액트 ]
    1. 정의 : 페이스북 회사(메타)에서 개발한 자바스크립트 라이브러리(함수 집합)
        * 예] 좋아요 기능, 댓글쓰기 기능하면 페이지 전환해야 되는 불편함.
        * 즉] ********* 페이지 전환이 없는 기능 구현 ************
    2. 목적 : 
        1. 자바스크립트에서 부족한 UI 조작과 효율성 보완 
        2. 웹/앱 유연한 UI 제공
        3. 트렌디한 변화 빠르게 대응
    3. 특징
        1. 화면(HTML)을 컴포넌트(함수) 기반, 재사용 가능
        2. 가상 DOM : 가상(가짜) HTML 사용하여 성능 최적화
        3. ** 훅( Hooks ) ** 지원 : 함수형 프로그램밍 지원방식으로 다양한 기능 지원
    4. 사용처
        1. 웹개발 : React
        2. 앱개발 : React Native
        3. 웹/앱 개발 : React Native web
    5. 
        - SPA ( Single page Application )
            1. HTML 한장으로 구성, CSR( Client side Rendering ) 방식
            2. 클라아언트사이드렌더링이란 ? 사용자 브라우저에서 화면 전화
            3. 장점 : 빠르다, 서버부담적다, 단점 : 최초로딩느리다, 검색엔진(SEO) 불리함(네이버(구형이라)에서 검색안됨)
            예시] REACT( web2 ), 
            + 단점보완 : 서버사이드렌더링을 고집하게 하기 위해 NEXT.JS 도입한다.

        - MPA ( Multi page Application )
            1. HTML 여러장으로 구성, SSR( Sever side Rendering )방식
            2. 서버사이드렌더링이란 ? 서버가 제공하여 사용자브라우저에서 화면 전환
            3. 장점 : SEO 장점,빠른초기로딩, 필요한 데이터만 사용/ 단점 : 새로고침(깜빡거림), 서버부담크다.
            예시] JSP( web1 )
